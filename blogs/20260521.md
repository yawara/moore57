### `Foundations/LinearAlgebra/PowPrimeTrace.lean` (新規)

主結果: 任意の ℚ-線形自己準同型 `f : W →ₗ[ℚ] W` (有限次元) について
`f^p = 1` (p prime) なら `trace(f) ∈ ℤ`。

戦略:
- `cyclicProjection f p = (1/p) • (1 + f + f² + ... + f^(p-1))` を定義
- `s² = p·s` (cyclicSum_sq) より `cyclicProjection` は idempotent
- `range(proj) = ker(f - 1)` (固定部分空間), 上で `f` は恒等
- `ker(proj) = ker(cyclicSum f p) = ker(aeval f cyclotomic_p)` (p prime)
- DirectSum trace decomp 経由で `trace = dim(range) + trace(f|_{ker})`
- 上で既存 `trace_package_of_cyclotomic_prime_aeval_eq_zero` を適用:
  `dim(ker) = (p-1)·γ`, `trace(f|_{ker}) = -γ`
- 合計: `trace(f) = dim(range) - γ ∈ ℤ`

これは既存 `exists_int_trace_of_involutive` (p = 2) の **prime-order 一般化**。

### `Moore57Graph/Aut/TraceIntegrality.lean` 

新規 `aut_pow_prime_E7_trace_int`: σ が prime 位数 p の自己同型なら
`Matrix.trace (E7Matrix Γ * permMatrix σ) ∈ ℤ`。

Mathlib `Matrix.toLin'_pow` で `(permMatrix σ ^ p).toLin' = (permMatrix σ).toLin'^p`
を bridge にして、E7-range への restrict が prime-order を保つことを示し、
`exists_int_trace_of_pow_prime_eq_one` に流す。

### Lem 12 p=3 starred 完全 unconditional 化

`lem12_no_p3_a0_one` (True-stub) を実証明に置換:
1. `aut_pow_prime_E7_trace_int` (p=3): `tr(E₇·P_σ) ∈ ℤ`
2. `lem3_a1_mod_15`: `a₁ ≡ 7·a₀ + 5 (mod 15)`
3. `a₀ = 1`: `a₁ ≡ 12 (mod 15)`
4. `lem12_p3_a1_eq_zero` (no-triangle): `a₁ = 0`

これは **論文の Theorem 3 (Curtis–Reiner) を完全に経由しない**

### 観察

- **Cyclotomic 機構の最小単位**: 全 Galois 理論を建てる必要はなく、既存の
  `CyclotomicPrimeTrace.lean` (γ 個の Φ_p ブロックで trace = -γ) + 
  Bool-indexed `IsInternal` direct sum decomp で十分。
- **Polynomial restriction の壁**: `aeval (f.restrict h) q = 0` を示すために
  `Polynomial.aeval` の restrict 互換性を直接示すのは複雑。回避策として
  `cyclicSum (f.restrict h) p` を直接定義し、各 power について 
  `restrict_pow_apply` で underlying 一致を示す方式が clean。
- **Lake build 状態**: 自分の 3 ファイル (PowPrimeTrace, TraceIntegrality, 
  Lemma12_PrimeOrder) は clean ビルド。ただし他セッションで追加された 
  `Foundations/GraphTheory/AutSubgroup.lean` と 
  `Foundations/GroupTheory/SmallGroup625_12.lean` は別途エラーあり 
  (Moore57.lean に import 済み)、当作業の範囲外。

`a₀(σ) ≤ a₀(σ³) = 1` 単調性で `(?, 1)` 列全体が排除される。

  に限定。 これは B4.3 (Galois + generalized eigenspace) 案件で
  deferred-heavy。
- **コード量**: 各 row 30-50 LOC で完了、ロードマップ見積もり (50-100 LOC)

- **`Representation` の type-dependent rewrite 落とし穴**: subrep の

  良かったかもしれない。 ただし今回は subrepresentation 経由の使い回し
  にトラブルがあったので、結果的に独立構築が clean。

### A1.1 orientation (Mathlib IsPGroup 分類)

**結論**: Mathlib に位数 81 (= 3^4) 群の分類は **無い**。
`Mathlib.GroupTheory.PGroup` の API:
- `IsPGroup p G` (card / orderOf 特性)
- `card_modEq_card_fixedPoints` (Sylow 系の基盤)
- `cyclic_center_quotient_of_card_eq_prime_sq`,
  `commutative_of_card_eq_prime_sq` (p² まで)
- `card_center_eq_prime_pow` (p^n の center 非自明性)

これらでは「位数 p⁴ 群の 15 isomorphism classes」は取れない。
A1.2 (uniqueness) には 15 群直接構築 (各 100-200 LOC) が必要、と
ロードマップに記録。

### E5.1: Sylow + Schur-Zassenhaus bridge

`Foundations/GroupTheory/SylowSchurZassenhaus.lean` 新規:

```lean
theorem Sylow.exists_complement_of_normal
    (P : Sylow p G) [(P : Subgroup G).Normal] :
    ∃ K : Subgroup G, Subgroup.IsComplement' (P : Subgroup G) K :=
  Subgroup.exists_right_complement'_of_coprime P.card_coprime_index
```

Mathlib `Sylow.card_coprime_index` (Sylow p-subgroup の order と index は
互いに素) と `Subgroup.exists_right_complement'_of_coprime`
(Schur-Zassenhaus 主形) を組み合わせるだけで、**正規 Sylow p-subgroup は
必ず complement (semidirect 因子) を持つ** という generic 結果に。

既存の `prop6_sylow5_normal` / `prop7_q*_sylow*_normal` /
`prop8_q*_sylow*_normal` / `thm7_card_*_sylow*_normal` の各 normal Sylow
成果を、この helper で semidirect product 構造に格上げできるようになった。
paper §5.1 の主張「Feit-Thompson も Philip Hall も Moore57 には不要 —
Sylow + Schur-Zassenhaus で十分」が Lean レベルで支えられる形に。
